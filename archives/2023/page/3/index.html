Hi，我又来给大家更新了。

这次的内容是，什么数据库好。

我用过三种数据库：MySQL,PostgreSQL,SQLite

如果让我推荐，我会推荐MySQL，好多网站程序对MySQL都非常的友好，比如WordPress，简直就是数据全放MySQL里面。只要数据库还在，网站程序扔了也罢。而且好多免费主机用的都是MySQL，对小白非常友好。

PostgreSQL不知道是不是我的环境有问题，刚开了个新库就快要8MB了，而且我用的网站程序就Z-Blog支持PostgreSQL，免费主机不支持。

SQLite也不是说不能用，但是你如果想迁移的话就会麻烦死掉，比如@你好的博客，因为一时的方便用SQLite，结果今天要迁移只能忍痛放弃以前的文章，要是企业还得了。。。

以上仅属于我的个人观点，有错误请在评论区指出，散会。

git clone 
cd cat

# 适用于CentOS/Ubuntu/Debian等主流Linux系统
# 只要系统架构是amd64，可以安装Python，就基本可以运行Cat
# 进入cat主程序目录，python main.py

  相信大家也看到了，这个栏目叫做浩炎专区。正如你所想象的那样，这是我专门为图形化运维大佬李浩炎 过渡至命令运维大佬所开辟的专区。如果你是纯小白，也可以进来看看，因为大佬让我写的详细点，还给我了一台服务器，我肯定要满足他。

  废话不多说，开始，走你！

  如果你被你的低配服务器气了个半死，但你又想建站，搭建一个简单的个人博客，我可以向你推荐LNMP一键安装包，因为它的最低要求是1HCPU，最少128MB的空闲内存，5GB的空闲硬盘，他的网址是：https://lnmp.org ，如果你是个运维大佬，可以跳过这个步骤；如果你不是，你可以按照我的来（小白千万别跳）：

  1. 把你的服务器重装成Debian或RHEL，Ubuntu要没被动过的。

  2. 你要有root权限，登录你的服务器。

  3. 敲脚本： 低配服务器：

    wget http://soft.vpser.net/lnmp/lnmp1.9.tar.gz -cO lnmp1.9.tar.gz && tar zxf lnmp1.9.tar.gz && cd lnmp1.9 && LNMP_Auto="y" DBSelect="2" Bin="y" DB_Root_Password="改成你的MySQL密码" InstallInnodb="y" PHPSelect="5" SelectMalloc="1" ./install.sh lnmp

普通服务器：

wget http://soft.vpser.net/lnmp/lnmp1.9.tar.gz -cO lnmp1.9.tar.gz && tar zxf lnmp1.9.tar.gz && cd lnmp1.9 && LNMP_Auto="y" DBSelect="4" Bin="y" DB_Root_Password="改成你的MySQL密码" InstallInnodb="y" PHPSelect="10" SelectMalloc="1" ./install.sh lnmp

高配服务器：

wget http://soft.vpser.net/lnmp/lnmp1.9.tar.gz -cO lnmp1.9.tar.gz && tar zxf lnmp1.9.tar.gz && cd lnmp1.9 && LNMP_Auto="y" DBSelect="5" Bin="y" DB_Root_Password="改成你的MySQL密码" InstallInnodb="y" PHPSelect="12" SelectMalloc="1" ./install.sh lnmp

然后你就可以坐在一边开会了，就算开上两年半也没问题(doge)。

如果他不跑了，你要好好看看有没有什么红色的代码，还有好多的那种，如果有，你的系统肯定被动过。

如果他没有，恭喜你，LNMP安装完成！

之后你可以输入reboot重启服务器了，下期视频教你如何玩转LNMP_Command
