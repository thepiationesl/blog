怎么样，我挂了三个节点，总有一个你进的来。

这里的管理们都非常的友善，丝毫不会乱骂人，什么不会都可以问管理。比如你的作业不会，那么你可以拍照上传给在线的管理，管理当场帮你解出来，再也不用怕被老师发现你抄作业了！

BB了那么多，我想我可以开始正题了。

  1. 你需要打开其中一个链接来到聊天室主页，选择一个聊天室，点击"JOIN CHAT"进入

  2. 你需要登录。如果你想保持匿名聊天，选右边的Guest，随便填写一下信息，点击"Login"进入聊天室

  3. 你也可以点击"Login"按钮下方的" Create an Account!"进行注册，注册完成以后可以直接在Member处登录，你的数据会被保留。

  4. 进入聊天室以后，点击"type a mesage"输入消息，点击小飞机图案发送你的消息。

  5. 当你看到一个想回复的消息时，将鼠标移动到那条消息，点击右边出现的三个点，点击Reply，再点击最底下的type a mesage输入消息，点击小飞机回复。

  6. 当你想清空聊天记录时，点击右边的options，Clear Chats，点击确定，即可你自己清空聊天记录。

  7. 当你遇见一个想要私聊的人时，可以点击左侧那个人的用户名，将鼠标移动到type a mesage进行私聊，或者点击那个人消息右边的三个点，选Start Chat即可私聊。

  8. 点击左下角你的名字时，你会发现你也可以自己创建一个聊天室，看不懂可以开翻译。

  9. 请文明聊天！

“折腾”路由器系列（1）

众所周知，H3C N12是最便宜的全千兆无线路由器。这款路由器给我的第一印象就是快。为什么说他快呢？你连上H3C_5G就知道了，只要您的手机支持5G频段的WIFI，速度快到飞起，搭配PT921网关，实测最高速度50Mbps，不过这对没钱人也是很友好的啦，毕竟100多的路由器，你也不能要求太多。如果不是经济限制，我会选择TP-LINK的路由器，可玩性比H3C N12高。

好啦好啦，回归正题，如果你手上还有其他过时的路由器，你可以进入后台看看他支不支持无线桥接（中继），如果支持的话，你可以继续往下看了。我这里以MERCURY MW325R为例来给大家演示，这种方法也可以让你合法的蹭邻居WIFI(WLAN)。

首先，我们要先激活H3C N12（已经激活的跳过）。进入后台，点击最底下的专业版，安全专区，防蹭网，勾选“自动将攻击设备加入禁止接入列表”，点击应用。再点击安全专区下的防攻击，将“丢弃攻击报文，不记入日志”改成“丢弃攻击报文，并记入日志”，最后把防范等级改成高级。如果你的周围还有一堆WIFI信号，那你可以把无线设置，智能网络设置，勾选“启用TDMA-Dig”，应用。之后进入简化版，重启设备。主路由器这边完工。

现在来到你的旧路由器这边。你记得先看看手机能不能收到主路由器的信号，最好在信号两格左右的地方插上旧路由器，然后按住reset键恢复出厂设置，MW325R、MW300R这些有好多灯的路由器灯全部亮了就可以放手了，MW313R、H3C N12这种只有一个灯的路由器按到灯开始闪烁了就可以放手了。之后连接旧路由器，设置完管理员密码后直接点跳过，来到高级设置，LAN口设置，LAN口IP设置改为手动，如果你没有故意去更改H3C N12的IP地址的话，那么你要把旧路由器的LAN口IP设置成192.168.124.252，252可以改成你喜欢也不会造成冲突的IP地址。改完以后点保存，重新连接旧路由器，输入你设置的IP，输入你设置的后台管理员密码，点高级设置，无线设置，WDS无线桥接，下一步，选择N12的WIFI名称，输入密码，然后旧路由器就开始和N12通信了。这个时候你如果是通过WIFI连接旧路由器的，那么你应该卡在正在连接。现在就让他卡着，你来到N12旁边，连接N12。如果你还是卡在这里，刷新一下，重新输入管理员密码，高级设置，DHCP服务器这边，打开DHCP，其他啥也别动，保存。然后到设备管理，重启路由器，重启路由器，重启完以后，来到旧路由器那边，连上旧路由器，访问moshujia.cn，如果进的去，恭喜你，桥接成功。

这个时候你的旧路由器变成了一个WIFI放大器，如果你的旧路由器以前带4个人都费劲，现在就看主路由器的心情了，如果主路由器厉害，可以带100个人，那么这台路由器也可以带100个人，因为他只负责信号放大，不负责处理网络，压力全在主路由器那边。在我这里N12带10台设备上网没啥压力，不知道你们那边怎么样，有任何问题可以在评论区提出来。

Hi，我又来给大家更新了。

这次的内容是，什么数据库好。

我用过三种数据库：MySQL,PostgreSQL,SQLite

如果让我推荐，我会推荐MySQL，好多网站程序对MySQL都非常的友好，比如WordPress，简直就是数据全放MySQL里面。只要数据库还在，网站程序扔了也罢。而且好多免费主机用的都是MySQL，对小白非常友好。

PostgreSQL不知道是不是我的环境有问题，刚开了个新库就快要8MB了，而且我用的网站程序就Z-Blog支持PostgreSQL，免费主机不支持。

SQLite也不是说不能用，但是你如果想迁移的话就会麻烦死掉，比如@你好的博客，因为一时的方便用SQLite，结果今天要迁移只能忍痛放弃以前的文章，要是企业还得了。。。

以上仅属于我的个人观点，有错误请在评论区指出，散会。

灵感来源：和@你好 一起商量什么操作系统好

大家好啊！我今天又来更新了。

好多人都在商量操作系统什么好，有的说Windows，有的说MacOS，还有人说Linux。我觉得，不管是什么操作系统，都好。如果你一定要让我推荐的话，那就接着往下看吧。

注意：我只推荐我用过的系统，其他的系统可以去@你好的博客自行研究

  1. Windows
    没错，我第一个推荐Windows系统。有人就问我，你不是天天都在骂Windows系统难用的不得了吗？的确，我一直在骂Windows系统难用，那是我没有电脑的时候。我当时玩的是远程桌面服务器，这个远程桌面服务器安装Windows系统卡的不得了，所以我才讨厌Windows。现在我有电脑了，电脑默认安装Windows系统，我也没有去重装他，因为在线下Windows异常流畅，那是因为远程桌面的话Windows杂七杂八的东西太多了，还要去处理RDP，对于没有显卡的服务器来说会大量消耗系统资源，这就导致我对Windows有这么大的误解。我其实也把电脑重装成Windows Server过，无奈驱动打不上，只能搞回Windows系统了。

  2. Linux Server
    这里我推荐Debian Server和CentOS Server。这是我用过的Linux服务器操作系统。请注意，这是给服务器用的操作系统，如果你想给你的电脑装上Linux Server，那就别想了。因为Linux Server只有Core组件，这会导致你的电脑资源大大地浪费，比如你想连接网络，只能把路由器拿过来，捅上网线，sudo apt update更新软件列表。他还没有图形界面，打个比方你想装QQ，人家QQ只能在图形界面上面装，你想p吃呢。

  3. Android
    第三个是安卓系统，当然我不是说手机的那个定制版安卓。这个安卓系统可以是Android x86，也可以是ChromeOS这类系统改的。这个我不多说了，电脑可以触屏当个平板用用，如果你不介意的话，插着鼠标也行，他可以模拟任务栏，就和TNT go差不多的体验。这个的话自己可以拿安卓手机试试，我写meilin CVBOS，对手机来说不好用，但是平板可以。你可以自己去百度一下，我以前都在用凤凰OS，可惜现在跑了，多好的系统啊！

  4. Linux Desktop
    没得说了，在Linux Server的基础上装个桌面环境。国内的话Deppn或者统信UOS，后者比前者更好用。

  5. BSD系列Unix系统
    这个不用我说了吧，FreeBSD和NetBSD还有好多BSD，Unix系统对我个人来说难用透了，比Linux Server还难用。OpenServer对电脑要求配置高而且还是给服务器用的，图形化软件更新大多数都在好几年前，你如果不是命令大佬或者运维不建议BSD。

最后，总结一下，这些仅为我的个人观点，有啥可议评论补充，欢迎评论！
